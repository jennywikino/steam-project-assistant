# TROUBLESHOOTING

## 常见问题

### 1. cmd 一关网页就不响应

原因：Streamlit 是本地服务，cmd 是服务进程。

解法：日常使用请双击 `启动工具.vbs`。调试时使用 `调试启动.bat`，这种模式下不要关闭 cmd。

### 1.1 关闭浏览器后后台服务还在

原因：关闭浏览器标签页不会关闭 Streamlit 后台服务，`8501` 端口仍可能被占用。

解法：正常退出请双击 `停止工具.bat`。需要确认服务是否还在时，双击 `查看状态.bat`。

### 2. 页面打不开

检查 Python、依赖、端口 8501。可以先运行 `停止工具.bat`，再重新启动。

V0.31e 起，启动脚本会自动检查 `8501` 端口；如果发现旧的项目初筛助手仍在运行，会尝试关闭旧进程后再启动。

V0.5.9 起，日常入口为 `启动工具.vbs`，它会后台调用 `tools/launchers/run_streamlit_hidden.bat` 并自动打开 `http://localhost:8501`。后台日志写入 `logs/streamlit.log`。

旧启动脚本已归档到 `tools/legacy_launchers/`。

### 2.1 启动提示 8501 端口仍被占用

原因：旧服务没有被自动清理，或端口被其他程序占用。

解法：

1. 双击 `停止工具.bat`。
2. 再运行 `启动工具.vbs`；调试时运行 `调试启动.bat`。
3. 如果仍失败，检查是否有其他程序占用 `8501`。

停止脚本只关闭占用 `8501` 端口的 PID，不会批量关闭所有 Python。

### 2.2 不确定后台服务是否还在运行

解法：

1. 双击 `查看状态.bat`。
2. 如果显示 PID，说明 `8501` 正在运行。
3. 如果显示“项目初筛助手未运行”，可以重新启动。

### 2.3 无窗口启动后看不到错误

原因：`启动工具.vbs` 会隐藏 cmd 窗口。

解法：

1. 查看 `logs/streamlit.log`。
2. 或双击 `调试启动.bat` 使用调试模式启动。
3. 如果日志里提示端口占用，先运行 `停止工具.bat`。

### 3. WPS 里 created_at 显示 ######

原因：列宽不够。

解法：看导出的 xlsx，不要直接看 CSV。

### 4. 中文乱码

检查 CSV 是否使用 `utf-8-sig` 编码。

### 5. 保存失败

检查 CSV 是否被 WPS 占用，关闭 WPS 后重试。

### 6. 竞品搜索结果不准

先用单关键词和英文关键词搜索，不要把所有中文词混在一起。

### 7. 换电脑启动失败

重新安装依赖；旧环境导出脚本已归档到 `tools/legacy_launchers/导出_环境信息.bat`。

### 8. 一键项目画像提示自动获取失败

原因：Steam 商店公开接口或页面可能临时不可访问，或者网络、地区、Steam 页面结构导致字段没有返回。

解法：

1. 确认输入的是 Steam 商店链接或纯数字 AppID。
2. 手动填写游戏中文名和补充关键词后再生成画像。
3. 画像中显示“未获取/需人工确认”的字段需要人工复核，不要当作最终判断。

本工具不使用需要登录、验证码、代理或 API Key 的方案，也不高频请求 Steam。

### 9. 画像报告没有生成

检查 `reports/profile/` 是否可写。如果项目文件夹被同步盘、杀毒软件或编辑器锁定，先关闭相关程序后重试。
